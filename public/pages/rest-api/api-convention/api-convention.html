API Convention

Introduction

When building an API, it's important that we follow some rules and conventions to make our code more maintainable and easy to understand.

HTTP Methods

HTTP methods are used to indicate the action that is needed to be performed.
Here there is a couple of methods such as:

GET: Requests data from a databse.
POST: This is typically used to insert new data into your database.
PUT: This replaces the whole row or data in the database.
PATCH: This replaces part of a existing data or row in a database.
DELETE: This deletes a specifc row in a database.

Here are some examples: 

            
   // GET method
   app.get('/api/users', function(req, res) {
     // insert code in here
   });
  
   // POST method
   app.post('/api/users', function(req, res) {
     // insert code in here
   });
  
   // PUT method
   app.put('/api/users/:id', function(req, res) {
     // insert code in here
   });

   // PATCH method
   app.patch('/api/users/:id', function(req, res) {
     // insert code in here
   });
  
   // DELETE method
   app.delete('/api/users/:id', function(req, res) {
     // insert code in here
   });

Resource Naming

Resource naming is very important when you are building an API. You should use nouns to name the resource and avoid using verbs.
Here are some examples:

            
      // Correct
      app.get('/api/users', function(req, res) {
        // insert code in here
      });
    
      // Incorrect
      app.get('/api/getUsers', function(req, res) {
        // insert code in here
      });

Response Format

The format of the response is important so that the client or other people using your api can parse the data correctly. The most common formats are JSON and XML.
you can also send a HTMl as a respone, which means it shows the page when you go to that end point.
Here are some examples:

            
  const users = [
      { name: 'John Doe', age: 25 },
      { name: 'Jane Smith', age: 30 }
    ];
  app.get('/api/users', function(req, res) {
    res.send({ data: users })
  });

Versioning

API versioning is a way to manage changes to your API over time. You can write the version before the name of your resource to indicate the version of your API.
Here are some examples:


    // Version 1 of the API
    app.get('/api/v1/users', function(req, res) {
      // insert code in here
    });

    // Version 2 of the API
    app.get('/api/v2/users', function(req, res) {
      // insert code in here
    });

Conclusion

By following these API conventions, you can make your code more maintainable and easy to understand. These conventions are widely used in the industry and can help you and your future colleagues to make a better API
