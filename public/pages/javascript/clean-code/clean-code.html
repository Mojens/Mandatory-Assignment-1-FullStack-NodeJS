Clean Code & Code Conventions

Introduction

Writing clean, and organized code is a very important part of being a javascript developer.
Following some code conventions can make your code more readable and easier to maintain,
while also making it more accessible to other developers who may work on the same codebase.
Here are some tips to keep in mind when writing JavaScript code:

Naming Conventions

It's important to use descriptive names for variables, functions, and classes.
This makes it easier to understand what the code is doing.
Here are some examples of good naming conventions:

                
    // Bad
    const name1 = "John";
    const name2 = "Doe";

    // Good
    const firstName = "John";
    const lastName = "Doe";

    // Bad
    function add(a, b) {
        return a + b;
    }

    // Good
    function add(firstNumber, secondNumber) {
        return firstNumber + secondNumber;
    }

Comments

Comments are a great way to explain what your code is doing.
However, it's important to not overuse them.
Comments should be used to explain complex code, or to explain why you're doing something.

    Example 2:
                
    // This function takes an array as an argument of numbers and returns the average
    function calculateAverage(numbers) {
      let sum = 0;
    
      // Loops through each number in the array and add it to the sum
      for (let i = 0; i < numbers.length; i++) {
        sum += numbers[i];
      }
    
      // Calculate the average by dividing the sum by the number of numbers in the array
      const average = sum / numbers.length;
    
      // Return the average value
      return average;
    }

Functions

Functions should be small and do one thing.
This makes it easier to understand what the function is doing.
Here are some examples of good functions:

                
    // Bad
    function add(firstNumber, secondNumber) {
        const sum = firstNumber + secondNumber;
        return sum;
    }

    // Good
    function add(firstNumber, secondNumber) {
        return firstNumber + secondNumber;
    }
