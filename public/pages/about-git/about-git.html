Git & Git-commands

Introduction

We use git to keep track of our code.
We use it to work on the same code with multiple people.
We use it to keep track of changes.

You need to install git on your computer to use it.
You can download git here.
Here is some examples of git-commands: 

Initializes a new Git repository
                    
      git init
                    
                
Clones a repository remotely
                    
    git clone [url]
                    
                
Adds a file to the staging area, so its ready to be committed
                    
    git add [file]
    // or
    git add .
                    
                
Commits the changes to the repository with a message
                    
    git commit -m [message]
                    
                
Displays the current status of the repository
                    
      git status
      // Example

On branch main
Your branch is behind 'origin/main' by 2 commits, and can be fast-forwarded.
  (use "git pull" to update your local branch)

Changes not staged for commit:
  (use "git add/rm ..." to update what will be committed)
  (use "git restore ..." to discard changes in working directory)
	modified:   app.js
	modified:   public/components/xxxx/xxxxx.html
	modified:   public/components/xxxx/xxxxx.html
	deleted:    public/pages/xxxx/xxxxx.html
	modified:   util/xxxxx.js

Untracked files:
  (use "git add ..." to include in what will be committed)
	public/pages/xxxxxxxxxx/

no changes added to commit (use "git add" and/or "git commit -a")
                    
So you can see that there are some changes that are not staged for commit.
So you can see modified files and deleted files, in that way you are sure that you are not missing anything.
                
Lists all branches in the repository
                    
      git branch
                    
                
Switches to an existing branch in the repository, and add -b to create a new branch
                    
    git checkout [branch]
                    
                
Merges the specified branch into the current branch
                    
     git merge [branch]
                    
                
Fetches and merges changes from a remote repository
                
      git pull
                
So in a case where you are working with someone else, and you want to get the latest changes, you can use this command.
            
Pushes the committed changes.
                
      git push
                
            
Displays a log of commits
                
      git log
