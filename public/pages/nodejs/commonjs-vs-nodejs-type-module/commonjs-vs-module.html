CommonJS vs type module

Introduction

When you are writing javascript code, there is different ways of importing and exporting modules.
The two different are CommonJS and type module. they both have the purpose of organizing and making it more reuasble and more readable.

CommonJS

CommonJS is based on sychronous approach. it uses require() to import modules and module.exports to export modules.

                
    // Importing module
    const module = require('./module.js');

    // Exporting module
    module.exports = {
        name: 'John Doe',
        email: 'JohnDoe@outlook.dk'
    }

It is important to note that the require() method is synchronous. This means that the code will not continue until the module is loaded.
This is not a problem if you are working with small modules, but if you are working with large modules it can be a problem.

Type module

Type module is based on asynchronous approach. it uses import to import modules and export to export modules.

                
    // Importing module
    import { name, email } from './module.js';

    // Exporting module
    export const name = 'John Doe';
    export const email = 'JohnDoe@outlook.dk'

    export default {
        name,
        email
    }
