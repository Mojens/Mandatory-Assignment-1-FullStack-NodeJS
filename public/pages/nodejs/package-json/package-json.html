Package.json

Introduction

Package.json is a file that contains information about your project. It contains information like the name of the project, the version, the author, the dependencies and more.
When you create a new project with npm init, npm will create a package.json file for you. 

                    
   npm init

Example

Here is an example of a package.json file. 

                    
    {
        "name": "Project Name",
        "version": "0.0.1",
        "description": "Project description",
        "type": "module",
        "main": "app.js",
        "scripts": {
          "test": "echo \"Error: no test specified\" && exit 1"
        },
        "repository": {
          "type": "git",
          "url": "git+https://github.com/User/repository-name.git"
        },
        "keywords": [
          "node",
          "javascript",
          "fullstack",
          "documentation",
          "nodejs",
          "express",
          "nodemon"
        ],
        "author": "Your name or username",
        "license": "MIT",
        "bugs": {
          "url": "https://github.com/User/repository-name/issues"
        },
        "homepage": "https://github.com/User/repository-name#readme",
        "dependencies": {
          "express": "^4.18.2",
          "express-session": "^1.17.3",
          "nodemon": "^2.0.21"
        }
      }
